FAQ

Common questions.

Anything else — our team responds within 24 hours.

Can I really start with just one use case?+
Yes — and it's the recommended approach. The platform infrastructure (integrations, security, RBAC) is deployed once. The first use case runs on that infrastructure. Adding a second use case reuses the same integrations — no new IT project, no new security review.
How do I know which use case to start with?+
The highest-ROI starting points are Contract Review & Compliance ($220K per lawyer per year) for legal-heavy organisations, Approval Automation for deal-velocity problems, and Email Intelligence for knowledge-worker productivity. During the demo, we'll help you identify the use case with the clearest before/after for your specific situation.
Are Work Credits shared across all use cases?+
Yes. Work Credits are universal across all active departments and use cases. If Legal has a quiet month, the credits carry over to HR. Nothing is siloed per tool or per use case.
What does IT need to do to enable a new use case?+
IT approves the API connections once during initial deployment. Adding a new use case that uses the same connected systems requires no additional IT work — you activate the module in the admin console. Adding a use case that requires a new system connection requires an additional API approval for that system only.
How is ZUUZ priced?+
Three layers: Platform Foundation (fixed monthly), Department Modules (flat fee per activated department), and Work Credits (charged only on successful task completion). No per-seat pricing. See the full pricing model.
