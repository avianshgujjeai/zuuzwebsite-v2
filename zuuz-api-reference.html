Authentication
The ZUUZ API uses OAuth 2.0 bearer tokens for user-context requests and API keys for server-to-server integrations. Service account tokens are available on Enterprise plans for zero-downtime automated access.
📡
Core endpoints
Agents: trigger and retrieve results. Documents: upload, retrieve, query. Workflows: trigger, monitor, retrieve. Analytics: pull productivity scores and usage. Approvals: submit, retrieve, and update.
🛡️
Webhooks
Real-time webhooks for agent completions, approval changes, workflow executions, and document events. HMAC-SHA256 signed for verification.
Quick start

First API call in 5 minutes.

1
Generate an API key
Admin console → Settings → API Keys → New Key
2
Authenticate
POST /v1/auth/token with your key to receive a bearer token
3
Trigger your first agent
POST /v1/agents/run with your document or data payload
4
Retrieve the result
GET /v1/agents/runs/{run_id} or use the webhook callback URL
# Trigger the Contract Risk Review Agent
curl -X POST \\
  https://api.zuuz.ai/v1/agents/run \\
  -H "Authorization: Bearer $TOKEN" \\
  -H "Content-Type: application/json" \\
  -d '{"agent_id":"contract-risk-review","document_url":"https://your-dms.com/contract.pdf"}'
# Response
{"run_id":"run_01J...", "status":"processing"}
FAQ

Common questions.

What can I build with the ZUUZ API?+
Internal tools that trigger ZUUZ agents from your own systems, custom dashboards pulling ZUUZ analytics, integrations with systems not covered by native connectors, and webhook processors that react to ZUUZ events in real time.
Is the API available on all plans?+
API access is available on Mid-Market and Enterprise plans. The 14-day trial includes read-only access. Webhook delivery and service account tokens are Enterprise features.
What are the rate limits?+
Starter: 100 requests per minute. Mid-Market: 1,000 per minute. Enterprise: custom — configured based on your expected volume.
Is there an SDK?+
ZUUZ SDKs for Python and Node.js are available in our GitHub repository. The REST API is also fully usable without an SDK — all endpoints accept and return standard JSON.
